## CLAUDE'S TECHNICAL ANALYSIS

### Systemic Issues (affect every page):

**1. Double AI messages** — every AI response renders twice across Home, Documents, and Tasks. The SSE stream handler is creating a message from accumulated tokens AND a separate message from the replace/done event. Root cause is in `chatStream()` in `api.js` and the event handlers in `Home.jsx`. This is the most visible bug in the app.

**2. Nothing persists** — chat history, tasks, documents, onboarding state all reset on page refresh. This is the single biggest Performance scoring risk. Chat needs localStorage persistence. Tasks need to call `loadTasks()`/`saveTasks()` on mount/change. Onboarding state needs to be read from Cosmos before the App.jsx gate decides whether to show onboarding.

**3. AI has no Pebble personality anywhere** — every AI endpoint (chat, decompose, summarise, explain, nudge, documents Q&A, tasks Q&A) responds in generic ChatGPT voice. The system prompt in `chat_service.py` needs stronger voice enforcement: lowercase, gentle metaphors, short sentences, one question per message, never dump multiple items without offering to help act on them.

**4. ###ACTIONS markers leak into chat** — raw JSON markers visible in Home chat. Backend regex in `chat_service.py` isn't catching all cases. Need double-strip: backend regex fix + frontend safety strip before rendering.

**5. "Chapter 5" hallucination** — seed/example data in Cosmos DB that the system prompt loads as task context. Must be cleared from the database.

**6. Action buttons don't execute** — "Add tasks" in Home and "Turn into tasks" in Documents only navigate without creating data. Must dispatch to Redux + Cosmos before navigating.

**7. Cross-page integration barely works** — Home can't create tasks. Documents can't push to Tasks. Tasks chat can't create tasks. The integration story is Pebble's core innovation pitch and it's the weakest part of the app right now.
